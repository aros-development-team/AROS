Input Text Fields

Because Flash 5 has some limitations on how input-text objects can be modified at runtime, some special case code exists to work around these limitations. Input text objects cannot be resized at runtime, and their font cannot be changed.

The XML compiler will treat specially <inputtext> (or <windowtext>) elements.

Collecting Input Text Fonts

The view compiler makes a pass over the XML source looking for <inputttext> elements. It tries to decide which font is being used, either directly or by inheritance of font properties from the enclosing view or class.

As a special optimization, input text which has a fixed width and height at compile time is compiled into a specific Flash inputtext resource with that width and height, using the routine SWFWriter. addCustomInputText(). This allows us to use the faster native Flash input text wrapping routines.

Thus, 

  <inputtext name="t2" width="200" height="64"  bgcolor="#afcccc"/>
However, adding the attribute resizable="true" will force the runtime to use the generalized (but slower) resizable input text resource and routines. 
  <inputtext name="t2" width="200" height="64" resizable="true" bgcolor="#afcccc"/>
