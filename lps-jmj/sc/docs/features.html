Script Compiler Features

The Script Compiler compiles JavaScript files into SWF byte codes.

Much of the work on the SC is independent of the SWF back-end — that is, it can be reused if we add another platform.

Also see Language-related Features.

Script Compiler

LaszloScript input
LaszloScript is defined in the Laszlo Application Language Reference. It's substantially similar to ECMAScript, with the addition of class and super constructs.
SWF output
Output can be incorporated into a SWF file. Generates SWF byte codes, and Breaks large programs into 32K chunks.
Error reporting
On a par with the Flash authoring tool. File name and line number are given.
Compiler speed
Within a factor of two of the Flash authoring tool.
Code quality
On a par with the Flash authoring tool.
Compiles the Laszlo library
(Including whatever ActionScript constructs are necessary).
Inline assembly
Library code needs support for inline assembly.
Synthetic code fragments
The interface compiler needs to parse script-valued attribute values and method bodies, and place them in a constructed parse tree that can be compiled (wih suitable error reporting at every step of the way). The compiler must expose parser and compiler APIs for this.
Warn/disallow Flash-specific code
Warn on references to _root, etc. (and provide alternatives solutions). So that application code will be portable. Turned off for library code.
ASV obfuscation
The ActionScript Viewer can't decompile the program.
